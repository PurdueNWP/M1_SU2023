Create Tutorial Templates

Running is code will take all the .ipynb notebooks in a folder and copy them to a template folder after removing all code from within the code chunks. It works for specific folders now, and is written in R and bash, but can be generalized further. It converts the .ipynb notebooks to .qmd files temporarily to do this, and .ipynb files require something in the code chunks, which for now is #. It also will redo relative filepaths so that images from the tutorials folder show up in these tutorial templates. Thank you Jacqueline Nolis for the regular expression code: https://twitter.com/skyetetra/status/1454472378243698694

## quarto: install the command line interface (CLI) from https://quarto.org

## list all tutorials
ipynb_files <- list.files(here("Week1"), pattern = ".ipynb")
ipynb_files <- tools::file_path_sans_ext(ipynb_files)

## for loop
for (f in ipynb_files) {  ## f = ipynb_files[3]
  
  ## convert to .qmd in template directory
  system(glue("quarto convert Week1/{f}.ipynb --output Week1-templates/{f}.qmd"))
  
  ## remove code from code chunks (leaving only #), redo image filepaths, overwrite the .qmd
  read_file(glue("Week1-templates/{f}.qmd")) %>%
    str_replace_all("```\\{python(.*)\\}[\\S\\s]*?```","```\\{python\\1\\}\n#\n\n```") %>%
    write_file(glue("Week1-templates/{f}.qmd"))
  
  ## convert template to .ipynb
  system(glue("quarto convert Week1-templates/{f}.qmd --output Week1-templates/{f}.ipynb"))
  
  ## delete temporary .qmd
  file.remove(glue("Week1-templates/{f}.qmd"))
  
}

Note: without the added # in the code chunks, they disappeared entirely in the .ipynb files, like this example. An additional space did not alleviate, so we’ll do comments for now.
